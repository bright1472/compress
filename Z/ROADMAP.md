# Titan Compress 迭代路线图 (V2.0+)

本文档记录了 Titan Compress 后续版本的核心演进方向，由于 Web 沙盒限制，我们将重点通过“极客式设计”解决闭环交互中的痛点。

## Phase 1: 流程自动化与“无感”导出 (PC 优先)

### 1.1 自动导出目录 (PC-Side Automation)
- **技术栈**：File System Access API (`window.showDirectoryPicker`)。
- **功能**：用户一次性指定输出文件夹，Titan 在处理完每一项任务后，自动将结果写入该目录，无需人工逐一点击下载。
- **闭环感**：实现类原生 App 的“输入 -> 自动处理 -> 输出”完整链路。

### 1.2 自动下载流水线 (Auto-Download)
- **功能**：为移动端或不支持文件夹访问的浏览器提供。开启后，每完成一个文件自动触发一次浏览器下载动作。

---

### Phase 2: 沉浸式结果报告与用户指引

### 2.1 任务结案勋章 (Achievement Dashboard)
- **视觉**：压缩结束后弹出高价值感的摘要页（Glassmorphism 风格）。
- **指标**：
    - **成就感**：累计节省空间（GB/MB）。
    - **效率感**：平均压缩速率、总耗时统计。
    - **减碳足迹**（可选）：极客风格的小彩蛋。
- **动态描述**：根据压缩率给出动态评价（如“极致压缩”、“视觉无损”）。

### 2.2 移动端“清理指引”
- **功能**：由于浏览器无法自动删除相册原文件，在摘要页提供具体的清理建议：
    - “Titan 已为你节省 1.2GB，你可以前往系统相册放心删除原视频。”

---

### Phase 3: 体验深度优化

### 3.1 移动端防休眠 (Keep-Alive Strategy)
- **技术**：Screen Wake Lock API。
- **功能**：在压缩超大文件时请求屏幕常亮，防止手机熄屏导致任务挂起。

### 3.2 智能分辨率缩放 (Smart Downscaling)
- **功能**：作为“超大文件加速器”。当检测到 10GB+ 或 4K 素材时，主动推荐降级至 1080P 或 720P，以获取 4-8 倍的编码提速。

### 3.3 批量重命名模板
- **功能**：支持配置变量（如 `${name}_compressed_${quality}`）自动生成文件名。

---

### Phase 4: 全能型静态资源优化 (图片压缩扩展)

借鉴 `slim-gui` 的专业级图片处理逻辑，将 Titan 进化为全能型资源压缩工具。

#### 4.1 PNG 有损量化引擎 (TinyPNG 级优化)
- **技术**：引入 `imagequant` (WASM)。
- **功能**：通过“调色板量化”算法将 32 位 RGBA 图像压缩至 8 位索引色彩。
- **目标**：在保持肉眼无损的前提下，将 PNG 体积减少 60%-80%。

#### 4.2 多模块图片处理架构
- **实现**：采用插拔式引擎设计（`ImageProcessor`），解耦不同格式的压缩逻辑：
    - **JPG 引擎**：基于原生 Canvas 或 MozJPEG WASM。
    - **次世代引擎**：支持 **AVIF** (ravif WASM) 和 **WebP**，提供最高压缩率。
    - **HEIC 适配**：支持 iOS HEIC 格式的自动转码与压缩。

#### 4.3 智能转换与批量归档
- **功能**：
    - **一键 WebP 化**：将全量图片资源统一转换为次世代 WebP 格式。
    - **Exif 信息处理**：提供保留或清除图片的经纬度、拍摄设备等元数据的开关。

---

> [!TIP]
> Titan 的核心价值在于 **“隐私与零门槛”**。V2.0 的目标是通过 API 组合技以及 WASM 带来的原生性能，在 Web 环境下尽可能模拟出顶级原生 App 的操作效率与压缩质量。
