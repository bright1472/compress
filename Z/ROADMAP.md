## Phase 0：商业化闭环（优先级最高）

当前已有账号 + 激活码体系，但缺少用户自助购买通路，是商业化最关键的缺口。

### 0.1 在线支付接入
- **支付宝 / 微信支付**：国内主流收款
- **Stripe**：海外用户
- **激活码自动发放**：支付成功后自动生成并通过邮件/短信发送激活码
- **后端**：basic-backend 新增 `payment` 模块，对接支付回调、生成订单

### 0.2 定价页面 / 套餐展示
- 当前界面无"为什么要升级"的引导，用户感知模糊
- 在激活弹窗 / 达到限额提示处增加套餐对比（免费 5 次 vs 无限次 pro）
- 可配合 LTD（一次性买断）做早期推广

### 0.3 账号管理
- 修改密码
- 忘记密码（邮件重置）
- 退出登录 / 踢出其他设备
- 删除账号（符合 GDPR/个人信息保护法）

### 0.4 邮箱注册 + 邮件通知
- 当前账号无邮箱绑定，无法做密码找回
- 购买成功、激活码发放均需邮件通知

## Phase 1：用户体验闭环

### 1.1 自动导出目录（PC 优先）
- **技术**：File System Access API (`window.showDirectoryPicker`)
- **功能**：一次性指定输出文件夹，每个任务完成后自动写入，无需逐一点击下载
- **目标**："拖入 → 压缩 → 自动落盘" 全程零操作

### 1.2 批量 ZIP 下载
- 所有任务完成后，提供"打包下载"按钮，一次性下载 ZIP
- 解决移动端 / 无法使用文件夹 API 的浏览器场景

### 1.3 任务结案摘要
- 压缩完成后展示摘要卡片（累计节省空间、平均速率、总耗时）
- 根据压缩率给出动态评语（"极致压缩" / "视觉无损"）
- 移动端附加"前往相册删除原视频"清理建议

### 1.4 移动端防休眠
- **技术**：Screen Wake Lock API
- 压缩大文件时请求屏幕常亮，防止任务挂起

## Phase 2：压缩能力提升

### 2.1 PNG 有损量化（TinyPNG 级）
- **技术**：`imagequant` WASM
- 调色板量化算法将 32 位 RGBA 压缩至 8 位索引色
- 目标：在肉眼无损的前提下体积减少 60%-80%

### 2.2 HEIC / AVIF 输入支持
- iOS 相机默认 HEIC 格式，当前不支持输入
- 引入 libheif WASM 实现 HEIC 解码 + 重新压缩为 JPG / WebP

### 2.3 视频输出格式扩展
- 当前只输出 MP4，支持 WebM（VP9 / AV1）输出
- 新增 GIF 转 WebP / MP4（动图优化场景）

### 2.4 智能分辨率缩放
- 检测到 10GB+ 或 4K 素材时，主动推荐降至 1080P / 720P
- 给出速度预估（4-8 倍加速），用户确认后执行

## Phase 3：产品增长工具

### 3.1 分享 / 嵌入
- 生成压缩前后对比链接（端对端加密，24h 有效）
- 可作为引流入口

### 3.2 批量重命名模板
- 支持变量（`${name}_compressed_${quality}`）自动生成文件名

### 3.3 用量仪表盘（账号中心）
- 用户可查看本月压缩次数、节省空间、订阅到期日期
- 增加订阅续费入口

### 3.4 推荐/返利系统
- 分享专属推荐码，双方各得 N 次免费压缩
- 低成本拉新的常见手段

---

## Phase 4：企业 / API 版

### 4.1 REST API 接入
- 面向开发者 / 企业批量压缩场景
- API Key 鉴权，按量计费
- 提供 SDK（Node.js / Python）

### 4.2 私有化部署
- Docker Compose 一键部署整套（前端 + 后端 + MongoDB）
- 面向有数据合规要求的企业客户

---

> **核心判断**：Phase 0（商业化闭环）是最高优先级，没有支付通路就无法形成变现。Phase 1 是留存的关键，减少用户操作摩擦。Phase 2-4 是竞争壁垒，边做边迭代。
